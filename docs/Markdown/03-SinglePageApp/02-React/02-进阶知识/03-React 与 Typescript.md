**2 笔记**
* TS使用场景  
    **为何需要TS**
    > * 解决问题：JavaScript代码不规范；项目协同及维护效率偏低；

    **TS基础**
    > * 概念：TS是JS的一个超集，主要提供了类型系统和对ES6的支持，它由Microsoft开发，代码开源与GitHub上。
    > * 使用场景：生命周期较长，较稳定的大型项目。

    **`state`与`props`相同之处**

    **`state`与`props`不同之处**
