**2 笔记**
* Redux简介  
    **Redux是一个数据管理框架，经常和React搭配运用**
    **由于React写大型复杂应用遇到困难(代码结构、组件通信问题)，2014年Facebook提出Flux架构的概念**
    **2015年，Redux出现，将Flux与函数式编程结合在一起**

* Redux型优势和不足  
    **最大优势**
    > * 自带规范，将数据管理定义为三个核心概念  
    >> * store  
    >> * reducer  
    >> * action  

    **不足**
    > * 概念规范较多，学习成本高  
    > * 另一数据管理框架Mobx简单自由，但是缺乏规范，两者分别适用于不同的场景
